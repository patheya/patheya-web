# Content Guide - Patheya Technologies Website

This guide helps maintain consistent, high-quality content across the Patheya Technologies website.

## Brand Voice & Tone

### Voice Characteristics
- **Professional but approachable** - Expert without being intimidating
- **Clear and concise** - No jargon unless necessary
- **Confident** - We know our craft
- **Client-focused** - Always about solving their problems

### Tone Guidelines
- Use active voice: "We build" not "Applications are built by us"
- Be specific: "25+ projects delivered" not "Many projects"
- Show, don't just tell: Use case studies and examples
- Avoid hype: Let results speak for themselves

## Writing Guidelines

### Headlines
- **H1:** One per page, clear and descriptive (50-60 characters)
- **H2:** Section headings, descriptive and scannable
- **H3:** Subsections, support H2 content

Example:
```markdown
# Custom Software Development Services (H1)
## Our Development Process (H2)
### Discovery & Requirements Analysis (H3)
```

### Body Content
- **Paragraphs:** 2-4 sentences maximum
- **Sentences:** Clear and direct (15-20 words average)
- **Lists:** Use bullet points or numbered lists for scannability

### Call-to-Action (CTA) Text
Good CTAs:
- "Schedule a Consultation"
- "Discuss Your Project"
- "View Our Work"
- "Get in Touch"

Avoid:
- "Click Here"
- "Learn More" (too vague)
- "Submit" (too generic)

## SEO Content Guidelines

### Keywords to Use
**Primary Keywords:**
- Software development company Pune
- Mobile app development India
- Cloud DevOps services
- ReactJS development company

**Secondary Keywords:**
- Custom software development
- React Native app developers
- AWS cloud migration
- Enterprise software solutions

### Keyword Usage
- Include primary keyword in H1
- Use related keywords in H2 headings
- Mention naturally throughout content
- Don't force or over-optimize

### Meta Descriptions
- 150-160 characters
- Include primary keyword
- Compelling and action-oriented
- Unique for each page

Example:
```
Expert software development services for web, mobile, and cloud solutions.
React, Node.js, AWS expertise. 25+ successful projects delivered. Get in touch today.
```

## Page-Specific Content Guidelines

### Homepage

**Hero Section:**
- Headline: Clear value proposition (8-12 words)
- Subheadline: Expand on headline (15-25 words)
- CTA: Primary action button

**Services Section:**
- Service title (3-5 words)
- Description (2-3 sentences)
- Key benefit or stat

**Statistics:**
- Use real numbers
- Include "+" for ongoing counts (15+ clients)
- Keep labels short

### Service Pages

**Required Sections:**
1. Introduction (what we offer)
2. Technologies used
3. Process/methodology
4. Use cases and applications
5. Why choose Patheya for this service
6. FAQ (5-8 questions)
7. CTA

**Word Count:** 800-1200 words per service page

### About Page

**Story Arc:**
1. Who we are (present)
2. What we believe (mission/values)
3. Why we exist (vision)
4. What makes us different

**Tone:** More personal, show company culture

### Case Studies

**Structure:**
1. **Overview:** Client, project type, timeframe
2. **Challenge:** What problem existed (150-200 words)
3. **Solution:** How we addressed it (300-400 words)
4. **Results:** Outcomes and impact (150-200 words)
5. **Technologies:** What we used and why

**Length:** 800-1000 words per case study

## Accessibility in Content

### Alt Text for Images
- Describe what's in the image
- Keep it concise (125 characters max)
- Don't start with "Image of..."
- Include relevant keywords naturally

Examples:
- Good: "Team collaborating on software architecture using whiteboard"
- Bad: "Image of team meeting"

### Link Text
- Make link text descriptive
- Avoid "click here" or "read more"
- Link text should make sense out of context

Examples:
- Good: "View our mobile app development process"
- Bad: "Click here to learn more"

### Headings
- Use proper hierarchy (don't skip levels)
- Make headings descriptive
- One H1 per page

## Content Checklist

Before publishing any page, verify:

- [ ] H1 includes target keyword
- [ ] Meta description written (150-160 chars)
- [ ] All images have alt text
- [ ] Links use descriptive text
- [ ] Proper heading hierarchy (H1 > H2 > H3)
- [ ] Content is scannable (short paragraphs, lists)
- [ ] CTAs are clear and actionable
- [ ] No spelling or grammar errors
- [ ] Mobile readability checked
- [ ] Internal links to related pages added

### Capitalization
- Service names: Title Case
- Technologies: As per official styling (React, Node.js, AWS)
- Job titles: Title Case (Chief Technology Officer)

### Numbers
- Use numerals for stats: "15+ clients" not "fifteen clients"
- Spell out small numbers in prose: "Our four core services"
- Use commas for thousands: "1,000" not "1000"
